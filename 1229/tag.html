Q.태그란 무엇인가?

HTML 태그란 웹 페이지의 구조와 내용을 정의하기 위해 사용하는 표기법 브라우저에게 “이 부분은 제목이다”, “이건 문단이다”, “이건 이미지다” 같은 역할과 의미를 알려주는 표시됩니다

Q.태그의 기본구조는?

HTML 태그는 보통 **꺾쇠 괄호(<ㅠ >)**로 감싸서 작성되며, 시작 태그와 종료 태그로 구성

태그의 분류

Q.블럭요소란?

블록 요소는 웹 페이지에서 "한 줄 전체를 차지하는 요소" 즉, 블록 요소가 하나 나오면 자동으로 줄바꿈이 일어나고, 다음 요소는 아래 줄에 배치됩니다.

Q.블럭요소의 종류

div는 의미를 가지지 않는 일반적인 영역 구분 요소로, 레이아웃을 나누거나 여러 요소를 묶을 때 가장 많이 사용된다.
header, footer, nav, section, article는 문서의 구조와 의미를 명확히 표현하기 위해 사용되며, 검색 엔진과 접근성 측면에서도 중요한 역할을 한다.
h1부터 h6까지의 제목 태그가 있으며, 문서의 제목과 소제목을 계층적으로 표현한다.
p 태그는 하나의 문단을 나타내며, blockquote는 긴 인용문을 표시할 때 사용된다.
pre 태그는 작성한 형식 그대로 텍스트를 출력하는 블록 요소이다.
ul(순서 없는 목록)과 ol(순서 있는 목록)이 있으며, 이 안에 li 요소가 포함되어 실제 목록 항목을 구성한다. 이들 역시 기본적으로 블록 요소로 동작하여 목록 단위로 줄바꿈이 이루어진다.
table 요소 또한 블록 요소에 해당하며, 내부에 tr, th, td 등을 포함하여 행과 열 구조의 데이터를 표현한다.

Q.인라인요소란?

HTML 문서에서 줄을 차지하지 않고, 내용의 크기만큼만 공간을 차지하며 같은 줄 안에서 나란히 배치되는 요소
인라인 요소는 주로 텍스트의 일부를 꾸미거나 의미를 부여하는 데 사용되며, 블록 요소처럼 줄바꿈을 발생시키지 않는다.

Q.인라인요소의 종류?

a 태그는 하이퍼링크를 생성하는 데 사용되며, href 속성을 통해 링크 대상 URL을 지정한다.
span 태그는 특정 텍스트 부분을 스타일링하거나 스크립트로 조작할 때 사용되는 일반적인 인라인 컨테이너이다.
strong 태그는 텍스트를 강조할 때 사용되며, 기본적으로 굵은 글씨로 표시된다.
em 태그는 텍스트에 강조를 부여하며, 보통 기울임꼴로 표시된다.
img 태그는 이미지를 삽입하는 데 사용되며, src 속성을 통해 이미지 파일의 경로를 지정한다. alt 속성은 이미지가 로드되지 않을 때 대체 텍스트를 제공한다.
br 태그는 줄바꿈을 삽입하는 데 사용되며, 단독으로 사용되는 빈 태그이다.
code 태그는 프로그래밍 코드 조각을 나타내며, 보통 고정폭 글꼴로 표시된다.

제목태그

제목과 소제목을 표현하는 태그
h1부터 h6까지 총 6단계가 있으며, 숫자가 작을수록 더 중요한 제목을 의미하고 글자 크기도 기본적으로 더 크게 표시
h1은 문서 전체를 대표하는 최상위 제목으로 사용되고, h2부터 h6은 그 아래 단계의 소제목을 표현
제목 태그는 모두 블록 요소이기 때문에 한 줄 전체를 차지하며 자동으로 줄바꿈
결론적으로 문서의 계층 구조를 명확하게 표현하고 가독성과 접근성을 높이는 핵심 태그

문단태그

하나의 문단(paragraph)을 나타내는 태그
블록 요소이기 때문에 한 줄 전체를 차지하고, 앞뒤로 자동으로 줄바꿈과 기본 여백(margin)이 적용
p 태그는 단순히 줄을 바꾸기 위한 용도가 아니라, 의미 있는 문단을 표현하기 위한 태그이므로 줄바꿈만 필요할 때는 br 태그를 사용
텍스트 내용을 논리적인 단위로 구분하여 가독성과 문서 구조를 향상

텍스트태그

글자의 의미, 강조, 형식, 역할을 표현하기 위해 사용하는 태그
strong 중요한 내용을 강조하는 태그
span 의미를 부여하지 않고 텍스트를 묶는태그
em문맥상 강조를 나타내는 태그
b시각적으로 굵게 표시하는 태그
i 기울임을 표현하는 태그
mark는 강조 표시
small은 부가 설명이나 저작권 문구 사용
sup, sub 위첨자와 아래첨자를 표현하는 태그
br 줄바꿈을 위한 태그
q 짧은 인용문을 나타내는 태그
abbr 약어를 설명하는 태그
텍스트의 의미와 표현을 명확히 하여 가독성, 접근성, 검색 엔진 최적화에 도움

이미지태그

이미지를 표시하기 위해 사용하는 태그
img 태그는 src 속성으로 이미지 파일의 경로를 지정하고, alt 속성으로 대체 텍스트를 제공
의미 있는 이미지에는 반드시 alt 속성을 작성하는 것이 바람직하다
웹 페이지에 이미지를 삽입하고 접근성과 가독성을 높이기 위한 필수 태그

강조태그

텍스트의 중요성이나 강조의 의미를 표현하기 위해 사용하는 태그
strong 태그는 문장에서 특히 중요한 내용임을 나타내며, 브라우저에서는 기본적으로 굵은 글씨로 표시

인용태그

다른 사람의 말, 글, 출처가 있는 문장 등을 인용했음을 나타내기 위해 사용하는 태그
lockquote b태그는 긴 인용문을 표현할 때 사용되며, 블록 요소이기 때문에 한 줄 전체를 차지하고 문단 단위로 인용 내용을 표시
q 태그는 짧은 인용문에 사용되는 인라인 요소로, 문장 안에서 따옴표 형태로 표시
인용의 출처를 명시할 때는 cite 속성이나 cite 태그를 함께 사용할 수 있다. cite 태그는 책 제목, 논문명, 작품명 등 출처나 저작물의 제목을 나타내는 데 사용
외부 발언이나 문장을 명확히 구분하여 문서의 신뢰성과 구조를 높여주는 태그

목록태그

여러 항목을 나열하여 표현할 때 사용하는 태그
ul(unordered list) 태그는 항목의 순서가 중요하지 않을 때 사용되며, 각 항목은 li 태그로 표현된다. 브라우저에서는 기본적으로 점(●) 형태의 글머리 기호가 표시
ol(ordered list) 태그는 항목의 순서가 중요한 경우에 사용되며, 마찬가지로 내부에 li 태그를 사용한다. 기본적으로 숫자나 문자로 번호가 매겨져 표시
dl(description list) 태그를 사용하며, 용어나 개념을 설명할 때 적합
dl 안에는 설명할 용어를 나타내는 dt 태그와 그에 대한 설명을 담는 dd 태그가 포함
여러 정보를 의미 단위로 묶어 명확하고 정돈된 형태로 표현하기 위한 태그

구분선태그

내용의 흐름이나 주제가 바뀌는 지점을 구분하기 위해 사용하는 태그
hr 태그는 종료 태그가 없는 빈 태그이며, 블록 요소로 동작하여 한 줄 전체를 차지한다
문서의 주제나 흐름이 바뀌는 지점을 명확히 표시하여 가독성과 구조를 향상시키는 HTML 태그

주소태그

문서나 콘텐츠의 작성자 정보, 연락처 정보, 출처 정보 등을 나타내기 위해 사용하는 태그
address 태그는 블록 요소로 동작하며, 보통 문서의 하단이나 글의 끝부분에 배치된다
검색 엔진과 보조 기술이 연락처 정보임을 인식할 수 있도록 도와주어 접근성과 문서의 신뢰성을 높음
문서나 콘텐츠의 작성자 및 연락처 정보를 의미적으로 표현하는 태그

시맨틱태그

요소의 의미와 역할을 명확하게 전달하는 태그이다. 단순히 화면 배치를 위한 태그가 아니라, 해당 콘텐츠가 무엇을 의미하는지 브라우저, 검색 엔진, 보조 기술(스크린 리더)에게 알려주는 역할
header는 문서나 영역의 머리말을 나타내며 제목이나 로고, 검색창 등을 포함할 수 있다.
nav는 메뉴나 링크 모음처럼 주요 탐색 영역을 표현하는 데 사용된다.
main은 문서에서 가장 핵심적인 콘텐츠를 담는 영역으로, 한 페이지에 하나만 사용하는 것이 원칙이다.
section은 주제별로 콘텐츠를 묶는 영역으로, 문서의 구조를 논리적으로 나누는 데 도움
article은 독립적으로 의미를 가지는 콘텐츠(뉴스 기사, 게시글 등)를 나타낸다.
aside는 본문과 직접적인 관련은 없지만 보조적인 정보(사이드바, 광고 등)를 담는 데 사용된다.
footer는 문서나 영역의 마무리 부분으로 저작권, 작성자 정보, 관련 링크 등을 포함한다.
시맨틱 태그를 사용하면 문서 구조가 명확해져 가독성과 유지보수성이 향상되고, 검색 엔진 최적화(SEO)와 웹 접근성 측면에서도 큰 장점
웹 문서의 구조와 의미를 명확히 표현하여 이해하기 쉽고 접근성 높은 웹 페이지를 만드는 핵심 태그
